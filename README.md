## Core Principles

### Rule #1: ALWAYS PLAN FIRST
Every piece of code starts with a detailed plan. Planning-first isn't a coding technique - it's a thinking technique. Applies to code, reviews, testing, and system design.

### Rule #2: DOCUMENT EVERY ERROR
Errors are learning opportunities. Each one gets documented with prevention strategies.

### Rule #3: VERIFY EVERYTHING
Code isn't done when it runs - it's done when it proves itself correct.

### Rule #4: EXPERIENCE BEFORE SOPHISTICATION (Week 1 Discovery)
Build simple first, feel the pain, then add complexity. Pain-driven prioritization produces deeper understanding than complexity-driven development.
